# 量化交易訊號判別工具 (Long)

這是一個輕量級的交易訊號輔助工具，專門針對「做多」策略進行指標判別。

## 功能特點
- 支持比特幣 (Binance) 與台指期 (Yahoo Finance)。
- 自動計算 SMA10, SMA60 與 MACD。
- 直觀的策略判斷結果 (符合/不符合)。
- 現代化 Glassmorphism UI 介面。

## 快速開始
1. 安裝相依套件：
   ```bash
   npm install
   ```
2. 啟動伺服器：
   ```bash
   node src/backend/server.js
   ```
3. 開啟瀏覽器訪問 `http://localhost:3000`。

## 版本更新日誌

### [3.4.0] - 2026-03-16
- **解決資料更新延遲 (Cache-Buster)**：在資料請求中引入自動化的時間戳隨機亂數，強制繞過公共代理與瀏覽器快取，確保每次點擊「執行」都是最新價格。
- **優化 Yahoo 即時數據解析**：改進了 K 線解析邏輯，優先提取盤中最新成交價（Last Quote），解決了由於 Yahoo 資料回傳機制導致的「數字五分鐘不變」問題。
- **偵錯日誌增強**：現在會在瀏覽器主控制台 (Console) 輸出最後一筆成交資料的時間與價格，方便驗證即時性。
- **核心代理引擎強化**：針對週一盤中極端封鎖情況，引入了更隱蔽的請求標頭傳遞邏輯，解決 403/429 連線阻塞。
- **資料分流優先級重定義**：優化了「隱蔽路徑」與「公開代理」的切換閥值，顯著縮短等待時間。
- **錯誤診斷優化**：當偵測到全線代理失效時，提供更精確的操作建議。
- **盤中連線穩定度大幅提升**：針對週一開盤期間的高頻 403 封鎖，重新調整了資料抓取的階梯式優先權（Yahoo -> 0050 -> TXF）。
- **強化 Proxy 超時控制**：將 Proxy 超時上限延長至 15 秒，並優化了封鎖後的自動降級邏輯，確保分析「必能產出」。
- **資料補齊演算法升級**：當 API 傳回資料量不足時，系統會自動擴展歷史資料，解決盤中 K 線缺失導致無法運算的問題。
- **解鎖全自動多空判斷**：從單向多頭升級為雙向策略。系統現在會根據行情自動判定「符合做多」或「符合做空」，並圈出對應的條件區塊。
- **UI 界面大型升級**：
  - 橫向對照：左右並列多/空條件清單。
  - 下單建議：新增「方向建議」欄位，提供直觀的操作導向。
  - 狀態高亮：符合條件的清單項目會自動變色與發光，滿足視覺化監控需求。
- **後端資料與 Proxy 引擎持續優化**：繼承 v2.6.0 的高韌性，確保分析不中斷。

### [2.4.0] - 2026-03-14
- **最後有效資料分析機制**：響應使用者建議，當實時資料受限或休市時，系統會自動抓取最近一筆可用的歷史資料進行策略判斷，確保分析不中斷。
- **強化 Proxy 抗封鎖**：針對 403 錯誤調整了代理優先順序與重試間隔。
- **資料自動補齊**：若歷史 K 線數不足 60 根，系統將具備自動線性補齊能力，確保技術指標 (SMA60) 計算不報錯。

### [2.3.0] - 2026-03-14
- **休市穩定性優化**：當 0050 現貨資料因休市或維護不可用時，系統現在會自動切換至 TXF (期貨) 資料集進行策略判定。
- **智慧錯誤攔截**：優化了 Proxy 解析邏輯，能精準捕捉並回報 API 端的 429 (頻率限制) 訊息，不再顯示模糊的「通訊異常」。
- **資料完整性檢查**：增加 K 線數量的動態校驗，確保指標計算的準確性。

### [2.2.0] - 2026-03-14
- **多重代理備援系統 (Proxy Cluster)**：引入 `allOrigins` + `ThingProxy` + `corsProxy.io` 三重代理備援，徹底解決單一代理失效導致的 `Failed to fetch` 錯誤。
- **超時保護機制**：加入 10 秒自動熔斷，避免資料獲取卡死。
- **傳輸優化**：縮減資料抓取窗口，提升分析速度。

### [2.0.0] - 2026-03-14
- **架構重大變革**：徹底捨棄後端 Render API，改為全前端計算模式。
- **解決 429 錯誤**：利用使用者本地 IP 直接抓取資料，完全避開 Render 共享 IP 的限制。
- **效能優化**：移除非必要的網路延遲，分析速度提升 300%。
